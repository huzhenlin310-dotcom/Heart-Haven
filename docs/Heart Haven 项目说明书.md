# Heart Haven（心栖）项目说明书

## 1. 项目概述

Heart Haven（心栖）是一款面向个人正念练习与自我关怀的轻量级 Web App。项目通过“选择音频、开始冥想、保存记录、查看趋势”的完整闭环，帮助用户在日常生活中建立可持续的冥想习惯，并用温和、私密的方式记录自己与重要他人之间的关怀连接。

项目不是医疗诊断或心理治疗工具，不替代专业心理咨询、医疗建议或危机干预服务。它的定位是一个低门槛、可离线使用、以本地隐私为优先的个人陪伴型应用。

## 2. 项目核心目的

本项目的核心目的是帮助用户在压力、疲惫或情绪波动时，快速进入一段安静、可持续、可记录的自我关怀练习。

具体目标包括：

- 降低冥想练习的开始门槛，让用户通过一个清晰的入口快速开始。
- 提供音频、倒计时、呼吸提示和当下引导，帮助用户完成一次完整练习。
- 将每次冥想自动沉淀为“旅程记录”，让用户看到自己的持续投入。
- 通过统计和打卡日历，增强用户对长期习惯的感知。
- 通过“关怀记录”模块，帮助用户记录重要的人和收到的善意，强化积极关系体验。
- 优先保护隐私，所有个人记录默认仅保存在用户当前设备。

## 3. 目标用户

Heart Haven 适合以下用户群体：

- 希望开始冥想，但不想使用复杂工具的新手用户。
- 有正念、慈心冥想、自我关怀练习需求的个人用户。
- 希望记录冥想频率、累计时长和连续打卡情况的习惯养成用户。
- 希望保存“重要的人”和日常善意片段的用户。
- 重视隐私、希望数据不上传云端的用户。

## 4. 核心功能

### 4.1 首页与快速开始

首页是用户进入应用后的主要入口，突出“开始冥想”这一核心动作。

当前支持：

- 主按钮进入音频选择页面。
- 使用上次选择的常用音频快速开始冥想。
- 展示最近一次冥想记录，包括练习类型、音频名称、时长和完成时间。
- 顶部标识“本地保存”，明确传达隐私设计。

### 4.2 音频选择

用户可以在开始冥想前选择本次练习使用的音频。

当前音频分类包括：

- 自我关怀正念冥想：当前主要开放分类，包含多段完整冥想音频。
- 自然白噪音：包含雨声等自然环境音。
- 舒缓音乐：作为后续扩展分类预留。

音频选择功能支持：

- 分类展开与收起。
- 显示音频标题和时长。
- 记住用户上次选择的音频，供下次快速开始使用。

### 4.3 冥想过程

冥想页面负责承载一次完整练习。

当前支持：

- 音频播放、暂停、继续和结束。
- 根据音频真实时长展示倒计时。
- 展示已进行时间。
- 呼吸节奏提示，按“吸气 / 呼气”切换。
- 根据练习进度显示当下提示文案。
- 音频无法播放时提供静默冥想计时。
- 用户离开冥想页面时进行确认，避免误操作丢失进度。
- 练习满 3 分钟后，结束时会自动保存为一条冥想记录。

### 4.4 冥想结果与旅程票据

一次冥想完成后，系统会生成一张“旅程票据”，作为用户本次练习的记录。

票据内容包括：

- 第几次旅程。
- 冥想类型。
- 音频名称。
- 冥想时长。
- 完成时间。
- 一句温和的鼓励文案。
- 收藏状态。

结果页支持：

- 收藏或取消收藏本次记录。
- 再冥想一次。
- 前往统计页面查看长期趋势。

### 4.5 冥想统计

统计页面帮助用户理解自己的练习情况。

当前支持的统计范围：

- 日
- 周
- 月
- 年
- 总计

统计指标包括：

- 累计冥想时长。
- 累计冥想天数。
- 累计冥想次数。
- 平均每次冥想时长。

页面还包含打卡日历，用于展示本月冥想日期，并计算：

- 当前连续打卡天数。
- 最长连续打卡天数。
- 本月打卡天数。

### 4.6 关怀记录

关怀记录模块用于记录用户生活中重要的人，以及他们带来的善意片段。

用户可以添加关怀对象，信息包括：

- 名字
- 性别（可不填写）
- 关系
- 生日

每个关怀对象支持记录简短的善意内容，例如一句话、一个动作或一次帮助。每条记录会保存创建时间，并计入该对象的关怀分数。

列表页支持：

- 按最近记录时间排序展示关怀对象。
- 快速添加 10 字以内的关怀内容。
- 展示最近一条关怀记录。
- 展示部分历史记录。
- 当记录较多时进入专属详情页。

详情页支持：

- 查看某个关怀对象的全部记录。
- 修改生日。
- 继续追加关怀内容。

### 4.7 设置与本地数据

设置页主要向用户说明数据保存方式、音频来源和版本信息。

当前支持：

- 查看已保存的冥想记录数量。
- 查看已保存的关怀对象数量和关怀记录数量。
- 查看当前音频数量和分类数量。
- 查看更新记录。
- 清除全部冥想记录。

数据存储策略：

- 冥想记录保存在浏览器本地。
- 关怀对象和关怀内容保存在浏览器本地。
- 常用音频选择保存在浏览器本地。
- 应用默认不将个人记录上传到服务器。

### 4.8 PWA 与离线能力

Heart Haven 支持作为 PWA 使用，具备类 App 的使用体验。

当前能力包括：

- 可配置为独立窗口启动。
- 支持移动端竖屏优先体验。
- 使用 Service Worker 缓存应用基础资源。
- 缓存图标、更新文件、页面入口和音频资源。
- 网络不可用时，已缓存页面可继续打开。
- 支持检测新版本，并提示用户刷新使用最新内容。

## 5. 用户使用流程

### 5.1 冥想主流程

1. 用户打开 Heart Haven。
2. 点击“开始冥想”。
3. 选择本次想使用的音频。
4. 进入冥想页面，点击开始播放。
5. 跟随音频、倒计时、呼吸提示和当下引导完成练习。
6. 结束练习后，系统保存旅程票据。
7. 用户可收藏记录、再次冥想或查看统计。

### 5.2 快速冥想流程

1. 用户打开 Heart Haven。
2. 点击首页的常用音频快捷入口。
3. 应用直接使用上次选择的音频开始冥想。
4. 完成后自动保存记录。

### 5.3 关怀记录流程

1. 用户进入“关怀”页面。
2. 添加一个重要的人。
3. 填写关系、生日等基础信息。
4. 在该对象下记录收到的善意。
5. 后续可进入详情页查看所有记录。

## 6. 客户价值

Heart Haven 对客户的价值主要体现在以下方面：

- 产品定位清晰：聚焦正念冥想、自我关怀和日常善意记录，不做复杂泛心理工具。
- 使用门槛低：核心操作路径短，用户可以快速进入练习。
- 有长期留存机制：通过旅程票据、统计、打卡和收藏记录增强持续使用动力。
- 隐私表达明确：本地保存策略降低用户对敏感记录上传的顾虑。
- 具备扩展空间：音频分类、关怀记录、统计维度和 PWA 能力都可以继续扩展。
- 适合静态部署：项目可作为纯前端应用交付，部署成本低。

## 7. 当前版本范围

当前版本为 1.5.1。

当前已实现：

- React + TypeScript + Vite 前端应用。
- 首页、音频选择、冥想、结果、统计、关怀、设置、更新记录等主要页面。
- 本地数据存储。
- PWA 安装清单。
- Service Worker 离线缓存。
- 更新记录和新版本提示。
- 适配桌面端和移动端的响应式界面。

当前未包含：

- 用户账号体系。
- 云端同步。
- 后台管理系统。
- 支付或会员功能。
- 医疗评估、心理量表或专业咨询服务。
- 多用户、多设备数据合并。

## 8. 技术交付说明

项目技术栈：

- React 19
- TypeScript
- Vite
- Tailwind CSS
- 浏览器 localStorage
- Service Worker
- Web App Manifest

部署方式：

- 项目构建后生成静态文件目录，可部署到 GitHub Pages、静态服务器或对象存储类平台。
- 音频、图标、图片等资源随静态文件一同发布。
- PWA 和离线缓存依赖浏览器支持。

本地数据说明：

- 用户数据与当前浏览器、当前设备绑定。
- 清除浏览器数据、更换设备或更换浏览器后，本地记录不会自动迁移。
- 如果后续需要多端同步，需要新增账号系统和后端数据服务。

## 9. 后续可扩展方向

后续可根据客户目标继续扩展：

- 增加更多音频分类，例如睡前放松、晨间正念、焦虑舒缓、专注练习。
- 增加情绪前后对比记录，帮助用户理解冥想前后的状态变化。
- 支持导出冥想记录和关怀记录。
- 增加云端同步和账号登录。
- 增加提醒功能，例如每日冥想提醒或生日提醒。
- 增加更多统计视图，例如月度报告、连续练习趋势和收藏回顾。
- 增加内容管理后台，方便客户自行维护音频和文案。

## 10. 总结

Heart Haven（心栖）是一个以“安静开始、温柔记录、长期陪伴”为核心的正念冥想 Web App。它通过简洁的冥想流程、本地化隐私记录、可视化统计和关怀记录模块，帮助用户在日常生活中建立自我照顾的习惯。

从客户视角看，该项目已经具备可演示、可部署、可继续扩展的产品基础，适合作为个人疗愈、正念练习、心理健康陪伴或生活方式类产品的早期版本。
